# 核心玩法机制

## 1. 当前玩法形态

当前项目不是传统菜单式 AVG，而是“命令驱动 + 角色响应 + 状态结算”的交互玩法。

玩家通过命令驱动系统，系统再通过：

- companion 内建命令
- 插件命令
- Agent 能力
- 状态与时间推进

生成结果。

## 2. 主循环

主循环位于 `GameEngine.ExecuteAsync`，当前流程是：

1. 解析命令前缀和参数
2. 判断命令是否可被当前路由处理
3. 执行 `CommandGate.BeforeExecuteAsync`
4. 进入 companion 命令或插件命令执行
5. 执行 `CommandGate.AfterExecuteAsync`
6. 输出结果

## 3. 当前玩法对象

### 玩家

- 发起命令
- 选择背景
- 指定当前 active character

### 角色

- 每个角色对应一个 runtime
- 可与玩家对话
- 可参与任务协作
- 可被其他角色咨询

### 世界状态

- 当前天数、分钟、地点
- 系统状态值
- 时间线事件

### 内容资源

- 背景
- 物品
- 插件提供的命令与 hooks

## 4. 当前可见玩法循环

### 开局

- 选择玩家名
- 选择背景
- 初始化 game state 与角色表

### 日常交互

- `/chat` 与当前角色对话
- `/switch` 切换当前角色
- `/characters` 查看角色列表
- `/task` 让多角色协作

### 系统行为

- 命令门控根据时间与事件控制是否允许执行
- 插件 hooks 可在聊天前后、睡眠前后等阶段介入
- 时间与状态系统共同影响玩法反馈

## 5. 当前玩法特点

- 玩家操作是命令式而不是按钮式。
- 对话不是纯文本拼接，而是由角色 Agent 与叙事渲染链共同生成。
- 任务协作已经具备最小多角色分发和汇总。
- 世界状态系统已经是玩法的一等公民，而非附加日志。
